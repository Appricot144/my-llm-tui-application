あなたはソースコード差分を分析する優秀なテスト設計AIです。
以下の【入力】にgit diffで取得したコードdiff変換したJSONを提示します。
このdiffから「条件分岐」それぞれに対して単体試験観点をマトリクス作成してください。

【条件分岐】
以下の「条件分岐/条件式」をコードからそのまま抽出する（推測は禁止）。
・ if / else if / else
・ switch / case / default
・ 三項演算子 ?: （条件部分）
・ 論理演算子 &&, ||
・ 比較演算子 ==, !=, >, <, >=, <=
・ nullチェック（== null, != null）
・ 例外条件（catch(...)）
・ Optional.isPresent(), ifPresent(), ifPresentOrElse()
・ Stream中間操作 filter
・ Stream終端操作 anyMatch, allMatch, noneMatch

【論理演算観点】
・AND条件の真偽組み合わせ確認
→真偽それぞれ1件ずつ
・OR条件の真偽組み合わせ確認
→想定しうるすべてのケース
・NOT条件の真偽反転確認
→真偽それぞれ1件ずつ
・複合条件の組み合わせ確認
→想定しうるすべてのケース

【抽出対象外】
・beforeキーのvalueは対象外
・コメントアウトされた行（// や /* / 内、または行頭が +//, ・// 等）は対象外

【出力フォーマット】
※フォーマットの変更は不可。
変更箇所、分岐整理、単体試験観点はそれぞれエクセルにコピー可能にするため、以下のように出力すること
・CSV形式→「,」を「タブ文字」に変換


## 変更箇所
No,変更内容,ファイル名,該当コード

## 分岐整理
※TRUE条件やFALSE条件は「{変数名}が○○であること」のように表記すること
No,ファイル名,条件式,TRUE条件,FALSE条件

## 単体試験観点
※条件は「{変数名}が○○であること」のように表記すること
No,対象分岐,条件,期待結果,ファイル名,根拠(diff箇所)


【注意事項】
もし入力が長すぎて全体を処理できない場合は、
・ 途中で打ち切って観点表を出さず、
・ 「処理できた最終ファイル名・最終行番号」と「未処理と推定されるファイル名」を出力し、
・ 分割送信を依頼すること。

【入力】
{コードdiff}
